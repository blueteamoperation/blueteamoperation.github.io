[ TLS DECRYPTION FORENSICS // WIRESHARK ]

LAB: CHAVE PRIVADA RSA & TRÁFEGO HTTP OCULTO
Cenário do Laboratório: Você possui uma captura de rede criptografada por SSL/TLS (rsa.cap) e a chave privada do servidor (rsa.key). Seu objetivo é configurar a chave no Wireshark para abrir a camada de aplicação e auditar as requisições HTTP trafegadas. Artefato : rsa.zip

[ PROCEDIMENTO DE DESCRIPTOGRAFIA NO WIRESHARK ]

  1. Abra o arquivo rsa.cap no Wireshark.
  2. Vá no menu superior em: Edit -> Preferences (ou Settings).
  3. No painel esquerdo, expanda Protocols e localize o protocolo TLS (ou SSL em versões mais antigas).
  4. Localize o campo RSA keys list e clique no botão Edit... (ou Pre-Shared-Keys de acordo com a versão).
  5. Adicione uma nova entrada apontando para o arquivo de chave privada rsa.key (Insira o IP do servidor, a porta 443, o protocolo http e selecione o arquivo da chave).
  6. Clique em OK / Aplicar. O Wireshark recarregará os pacotes mostrando as abas e dados em texto claro.

#01 - Volumetria de Artefatos de Mídia Filtro: http

Após aplicar com sucesso a chave rsa.key e filtrar o tráfego por http, quantas imagens totais existem sendo carregadas na captura?

#02 - Mapeamento de Recursos Inexistentes Filtro: http.response.code == 404

Filtrando o tráfego pelas respostas do servidor, quantos pacotes com o código de status HTTP 404 Not Found existem no dump?

#03 - Conexões Bem-Sucedidas Filtro: http.response.code == 200

Ainda na análise de métricas web, quantas respostas contendo o código de status HTTP 200 OK foram registradas após a descriptografia?
