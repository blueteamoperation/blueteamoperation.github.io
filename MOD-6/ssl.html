[ TLS DECRYPTION // PRE-MASTER SECRET LOG ]

LAB: AUDITORIA DE SESSÃO SSL KEYLOGFILE
Cenário do Laboratório: Você coletou um arquivo de log contendo os segredos de sessão TLS (Pre-Master Secret / SSLKEYLOGFILE) e precisa carregá-lo no Wireshark para descriptografar os pacotes de uma navegação protegida por HTTPS e inspecionar a volumetria das requisições web. Artefato : ssl.zip

[ PROCEDIMENTO DE DESCRIPTOGRAFIA NO WIRESHARK ]

  1. Abra o arquivo de captura de pacotes no Wireshark.
  2. Vá no menu superior em: Edit -> Preferences (ou Settings).
  3. No menu lateral esquerdo, expanda a árvore de Protocols e localize o protocolo TLS.
  4. Encontre o campo de configuração rotulado como (Pre)-Master-Secret log filename.
  5. Clique em Browse... e selecione o seu arquivo de log de chaves (SSLKEYLOGFILE).
  6. Clique em OK / Aplicar para que os pacotes sejam atualizados e o tráfego HTTP oculto seja revelado.

#01 - Volumetria de Sucesso da Aplicação Filtro: http.response.code == 200

Após carregar com sucesso o arquivo de chaves TLS e aplicar o filtro pelas respostas bem-sucedidas do servidor, quantos pacotes com o código de status HTTP 200 OK foram identificados?

#02 - Auditoria de Envio de Dados Filtro: http.request.method == POST

Inspecionando os métodos de requisição disparados pelo cliente, quantas ocorrências do método POST existem mapeadas na captura?

#03 - Mapeamento de Requisições de Leitura Filtro: http.request.method == GET

Por fim, ao filtrar apenas pelas requisições de download e leitura de recursos estruturais da página, qual é o total de requisições utilizando o método GET?
