📦 FERRAMENTA: tcpdump (CLI)

⚙️ 1. Parâmetros e Parâmetros de Configuração

  • -i any : Escuta o tráfego de todas as interfaces de rede ativas simultaneamente.
  • -i virbr0 : Determina a escuta em uma interface específica (Ex: virbr0).
  • -D : Exibe a listagem de todas as interfaces de rede elegíveis para captura de pacotes.
  • -n : Suprime a resolução de nomes de host (mantém os formatos puramente em IP numérico).
  • -nn : Desativa a resolução de nomes de host e nomes de serviços/portas (exibe IP e porta brutos).
  • -q : Ativa a saída resumida/silenciosa (Quiet output).
  • -t : Remove a impressão de marcações de tempo (timestamps) em cada linha de pacotes.
  • -tttt : Formata os timestamps na saída com a máxima legibilidade de data/hora para humanos.
  • -X : Apresenta os dados internos do payload tanto em formato Hexadecimal (HEX) quanto em ASCII.
  • -XX : Executa a mesma ação do parâmetro -X, mas inclui adicionalmente a exibição do cabeçalho Ethernet.
  • -v, -vv, -vvv : Modos progressivos de verbosidade para ampliar os metadados destrinchados do pacote.
  • -c [qtd] : Coleta apenas a quantidade especificada de pacotes e encerra a execução de forma programada.
  • -s [tamanho] : Estipula o tamanho do corte (snaplength) do pacote em bytes. Utilize -s0 para pegar o pacote integral.
  • -S : Imprime os números de sequência TCP em seu valor absoluto real em vez de numeração relativa.
  • -e : Habilita a extração e exibição do cabeçalho da camada Ethernet na linha de log.
  • -E : Executa a descriptografia de fluxos de tráfego IPSEC provendo uma chave criptográfica fornecida.
  • [QUALIFICADORES DE EXPRESSÃO PARA FILTRAGEM]
  • Tipo (Type): Filtros direcionados ao escopo do alvo. Opções nativas: host, net e port.
  • Direção (Dir): Determina o fluxo do tráfego. Opções nativas: src (origem) e dst (destino).
  • Protocolo (Proto): Isola a camada de transporte/rede. Opções nativas: tcp, udp, icmp, ah, etc.

🔍 2. Exemplos Práticos e Filtros de Flags TCP

  • Visualização de Tráfego Geral: tcpdump -i any
  • Monitorar Interface Fixa: tcpdump -i virbr0
  • Saída Bruta Forense Completa: tcpdump -ttttnnvvS
  • Buscar por IP Alvo: tcpdump host 192.168.122.131
  • Inspeção HEX de ICMP (Corte Zero/1 Pacote): tcpdump -nnvXSs 0 -c1 icmp
  • Filtrar Apenas por Origem: tcpdump src 192.168.122.131
  • Filtrar Apenas por Destino: tcpdump dst 192.168.122.14
  • Filtrar por Rede/Sub-rede Completa: tcpdump net 192.168.122.0/24
  • Filtrar por Porta Alvo (Ex: RDP): tcpdump port 3389
  • Isolar Protocolo ICMP: tcpdump icmp
  • Isolar Apenas Tráfego IPv6: tcpdump ip6
  • Filtrar Intervalo Amplo de Portas: tcpdump portrange 21-25
  • Filtrar por Tamanho de Pacote (Menor que 32B): tcpdump less 32
  • Filtrar por Tamanho de Pacote (Maior que 32B): tcpdump greater 32
  • Filtrar por Operador Matemático: tcpdump <= 102
  • Escrever Captura Direto para PCAP: tcpdump port 80 -w output
  • Ler Arquivo de Captura de Tráfego: tcpdump -r output.pcap
  • Operadores de Combinação Lógica: Usar and (&&), or (||) e exceção not (!).
  • Associação Lógica Avançada: tcpdump -nnvvS src 192.168.122.1 and dst port 4444
  • Agrupamento Complexo via Escopo: tcpdump 'src 192.168.122.84 and (dst port 4444 or 22)'
  • [ANÁLISE AVANÇADA DO BYTE OFFSET: tcp[13]]
  • Isolar Pacotes URG (Flag Urgent ativa): tcpdump 'tcp[13] & 32 != 0'
  • Isolar Pacotes ACK (Flag Acknowledge ativa): tcpdump 'tcp[13] & 16 != 0'
  • Isolar Pacotes PSH (Flag Push ativa): tcpdump 'tcp[13] & 8 != 0'
  • Isolar Pacotes RST (Flag Reset ativa): tcpdump 'tcp[13] & 4 != 0'
  • Isolar Pacotes SYN (Flag Synchronize ativa): tcpdump 'tcp[13] & 2 != 0'
  • Isolar Pacotes FIN (Flag Finish ativa): tcpdump 'tcp[13] & 1 != 0'
  • Isolar Conexão SYN-ACK Exata (Valor 18): tcpdump 'tcp[13] = 18'
  • Sintaxe Alternativa por Nome Nativo: tcpdump 'tcp[tcpflags] == tcp-syn' (Também aceita tcp-rst / tcp-fin)
  • Detecção de Anomalia (Ataque SYN+RST): tcpdump 'tcp[13] = 6'
  • Filtro Forense para HTTP GET puro (HEX ASCII): tcpdump 'tcp[32:4] = 0x47455420'
  • Identificar Banner SSH em qualquer porta: tcpdump 'tcp[(tcp[12]>>2):4] = 0x5353482D'

📦 FERRAMENTA: tshark (Wireshark CLI)

🖥️ 1. Comandos e Captura Ativa com tshark

  • Diretório Padrão de Instalação (Windows): C:\Program Files (x86)\Wireshark>tshark.exe
  • Listar Interfaces de Rede Disponíveis: tshark -D
  • Iniciar Captura Direta na Interface nº2: tshark -i 2
  • Capturar Tráfego com Tempo Limite (10 segundos): tshark -i 2 -a duration:10
  • Salvar Tráfego Capturado em Arquivo Externo: tshark -i 2 -w output_file.pcap
  • Aplicar Filtro de Captura Nativo (Exemplo DHCP): tshark -i 2 -f "port bootpc" -w DHCP_Only.pcap
  • Decodificar MPLS sobre UDP/GRE encapsulado (Exemplo Porta 51234 + ICMPv6):
    tshark -ni eth1 -Y "ip.addr==172.222.19.198 and ip.addr==172.222.19.199 and icmpv6" -d udp.port==51234,mpls
  • Extrair Host e User-Agent de Requisições HTTP em Tempo Real:
    tshark -i wlan0 -Y http.request -T fields -e http.host -e http.user_agent
  • Extrair Requisições DNS e seus Respectivos Endereços de Resposta:
    tshark -i wlan0 -f "src port 53" -n -T fields -e dns.qry.name -e dns.resp.addr
  • Extrair Metadados Completos do DNS (Time, IPs de Origem/Destino, Query e Response):
    tshark -i wlan0 -f "src port 53" -n -T fields -e frame.time -e ip.src -e ip.dst -e dns.qry.name -e dns.resp.addr
  • Capturar e Isolar Métodos HTTP POST que Contenham Senhas expostas:
    tshark -i wlan0 -Y 'http.request.method == POST and tcp contains "password"' | grep password

🔒 2. Análise Avançada de PCAP e Inspeção SSL/TLS

  • Análise Detalhada de Pacotes DNS a partir de Arquivo: tshark -nr input.cap -R "dns" -V
  • Mapear, Filtrar e Contabilizar Ranking de Conexões por User-Agent:
    tshark -r example.pcap -Y http.request -T fields -e http.host -e http.user_agent | sort | uniq -c | sort -n
  • Extrair Host, IP de Destino e URIs completas de Tráfego HTTP:
    tshark -r example.pcap -Y http.request -T fields -e http.host -e ip.dst -e http.request.full_uri
  • Filtrar Métodos HTTP GET de um pcap e salvar o resultado Limpo em outro Arquivo:
    tshark -2 -R "http.request.method==GET" -r HTTP_Traffic.pcap -w HTTP_Get.pcap
  • Exibir Estatísticas Globais da Hierarquia de Protocolos do Arquivo: tshark -r HTTP_Traffic.pcap -qz io,phs
  • Leitura Simples de pcap Sem Resolução de Nomes (Camadas 3 e 4): tshark -n -r nmap_sn.pcap ou tshark -nr nmap_sn.pcap
  • Isolar Conexões SSL/TLS do Tipo "Client Hello" (Handshake Type 1):
    tshark -n -r [arquivo] -Y 'ssl.handshake.type==1' -T fields -e ip.src -e ip.dst -e ssl.handshake.extensions_server_name
  • Extrair Destinos e Domínios SNI em Claro via SSL Handshake salvando em lista:
    tshark -r file.pcap -Y 'ssl.handshake.type==1' -T fields -e ip.dst -e tcp.srcport -e ssl.handshake.extensions_server_name | sed "s/\t/:/" > /tmp/ssi
  • Inspecionar Respostas de Certificados (Type 11) retornados pelo Servidor:
    tshark -r file.pcap -Y 'ssl.handshake.type==11' -T fields -e ip.src -e tcp.dstport -e x509sat.uTF8String -e x509ce.dNSName | sed "s/\t/:/" > /tmp/in
  • Auditar Conexões em Logs de Redes Anônimas (Tor pcapng): tshark -r tor.pcapng | findstr "Client Hello"
  • Mapear o Top 30 de Domínios mais Requisitados em Certificados (Campos DNSName):
    tshark -n -r [arquivo] -Y ssl.handshake.type==11 -T fields -e x509ce.dNSName | tr -s ',' '\n' | sort | uniq -c | sort -rn | head -30
  • Extrair URIs Únicas declaradas em Cadeias de Certificados Emissores:
    tshark -n -r [arquivo] -Y 'x509ce.uniformResourceIdentifier' -T fields -e x509ce.uniformResourceIdentifier | tr -s ',' '\n' | sort -u
  • Extrair Strings Gerais de Certificados (IA5, UTF8, Printable, Teletex, BMP, Universal):
    tshark -n -r [arquivo] -Y ssl.handshake.type==11 -T fields -e x509sat.IA5String -e x509sat.uTF8String -e x509sat.PrintableString -e x509sat.TeletexString -e x509sat.BMPString -e x509sat.UniversalString | tr -s ',' '\n' | sort -u
  • Extrair URIs de Certificados em Redes Tor para cruzamento com Bases de Ameaças:
    tshark -r tor.pcapng -Y ssl.handshake.type==11 -T fields -e x509ce.uniformResourceIdentifier
  • Extrair Nomes de Domínio de Certificados em Redes Tor para Inteligência de Ameaças:
    tshark -r tor.pcapng -Y ssl.handshake.type==11 -T fields -e x509ce.dNSName
  • Análise Hexadecimal Completa de Suítes de Cifragem confiáveis (Cipher Suites):
    tshark -r tor.pcapng -Y ssl.handshake.ciphersuites -Vx
  • Montar Assinaturas SHA1 Únicas para cada Cipher Suite mapeado no Client Hello:
    tshark -r [arquivo] -Y ssl.handshake.type==11 -T fields -e ssl.handshake.ciphersuite | sort -u | xargs -I {} sh -c 'echo -n {}" " && echo -n {} | sha1sum' | awk '{printf $2" "$1"\n"}'

📊 3. Filtros de Exibição Essenciais (Sintaxe Wireshark / tshark -Y)

  • [1. FILTROS DE PROTOCOLO]
  • tcp → Mostra todo o tráfego TCP.
  • udp → Mostra todo o tráfego UDP.
  • icmp → Mostra tráfego de diagnóstico (Ping / Traceroute).
  • http → Filtra requisições e respostas HTTP em texto claro.
  • tls || ssl → Isola tráfego seguro criptografado (HTTPS / TLS).
  • dns → Mostra todas as requisições e respostas do serviço DNS.
  • arp → Mostra requisições e mapeamentos do protocolo ARP.
  • ftp → Isola comandos e tráfego associado ao protocolo FTP.
  • smtp → Mostra tráfego de envio de e-mails (SMTP).
  • pop → Mostra tráfego de recebimento de e-mails (POP3).
  • [2. FILTROS DE ENDEREÇO IP]
  • ip.src == X.X.X.X → Filtra pacotes vindos de um IP de origem específico.
  • ip.dst == X.X.X.X → Filtra pacotes destinados a um IP específico.
  • ip.addr == X.X.X.X → Mostra qualquer pacote onde o IP corresponda à origem OU ao destino.
  • ip.src == X.X.X.X && ip.dst == Y.Y.Y.Y → Filtra o fluxo exato de uma origem para um destino específico.
  • !(ip.src == X.X.X.X) → Exclui completamente o tráfego vindo de um determinado endereço IP.
  • [3. FILTROS DE PORTA]
  • tcp.srcport == 443 → Filtra pacotes TCP vindos da porta de origem 443 (Geralmente HTTPS).
  • tcp.dstport == 80 → Filtra pacotes TCP destinados à porta 80 (HTTP).
  • udp.srcport == 53 → Isola respostas DNS vindas da porta de origem 53.
  • udp.dstport == 123 → Filtra pacotes NTP destinados à porta 123 (Sincronização de tempo).
  • tcp.port == 22 || udp.port == 22 → Mostra tráfego SSH em qualquer direção (origem ou destino).
  • tcp.dstport >= 1024 && tcp.dstport <= 65535 → Isola conexões destinadas a portas altas/efêmeras.
  • [4. FILTROS DE CONTEÚDO / PAYLOAD]
  • http contains "login" → Busca por pacotes HTTP que contenham a palavra textual "login".
  • dns.qry.name == "example.com" → Isola requisições DNS feitas para um domínio específico.
  • frame contains "malware" → Busca profunda por uma string/texto específico em qualquer camada do pacote.
  • ssl.handshake.extensions_server_name == "example.com" → Filtra o Handshake TLS procurando por um hostname específico no campo SNI.
  • [5. FLAGS TCP E ANOMALIAS (SOC / CAÇA DE AMEAÇAS)]
  • tcp.flags.syn == 1 && tcp.flags.ack == 0 → Detecta tentativas de varreduras de portas (TCP Port Scans) ou ataques de SYN Flood.
  • tcp.flags.reset == 1 → Isola encerramentos abruptos de conexão (Connection Resets).
  • tcp.flags.fin == 1 → Usado na detecção de varreduras do tipo FIN Scan.
  • tcp.flags.urg == 1 → Filtra pacotes que possuem a flag de urgência ativa.
  • tcp.analysis.retransmission → Localiza pacotes retransmitidos (Indício de perda de pacotes ou lentidão na rede).
  • tcp.analysis.flags → Destaca comportamentos incomuns ou corrompidos na camada TCP.
  • [6. FILTROS COMBINADOS E AVANÇADOS]
  • ip.src == X.X.X.X && tcp.dstport == 80 → Filtra tráfego HTTP vindo exclusivamente de uma origem determinada.
  • ip.addr == X.X.X.X && (tcp || udp) → Consolida todo o tráfego de transporte (TCP/UDP) associado a um único host.
  • tcp.port == 445 && ip.dst == 192.168.1.100 → Isola tráfego do protocolo SMB (compartilhamento de arquivos) direcionado a um host específico.
  • http.request.method == "POST" → Filtra e exibe apenas requisições HTTP do tipo POST (Envio de formulários/dados).
  • tcp.len > 1000 → Identifica pacotes de dados grandes, úteis na caça de transferências massivas ou exfiltração de dados.
  • frame.len > 1500 → Usado para detecção e análise de pacotes gigantes (Jumbo Frames).
  • [7. DICAS PRÁTICAS PARA OPERADORES DE SOC]
  • Seguir Fluxo (Follow Stream): No Wireshark, clique com o botão direito em um pacote TCP > Escolha Follow > TCP Stream para reconstruir a conversa inteira.
  • Regras de Colorização: Configure regras visuais para destacar flags críticas (SYN, RST, FIN) ou erros de resposta HTTP/HTTPS (Erros 4xx e 5xx).
  • Ferramentas de Estatística: Utilize o menu superior Statistics > Conversations ou Statistics > Endpoints para mapear volumetria e quem está conversando na rede.
  • Filtros Compostos: Sempre combine as camadas [IP + Porta + Protocolo + Flags] para fazer investigações cirúrgicas e reduzir os falsos positivos.
